🚀 NexusLedger: Node Shared KernelThe core infrastructure layer for the NexusLedger microservices ecosystem. This package handles resilient database transactions and reliable event broadcasting using the Transactional Outbox Pattern.🏗️ Architecture: Transactional OutboxTo ensure data consistency between PostgreSQL and Redpanda (Kafka), we use a dual-phase commit strategy:Local Transaction: The business logic and the event data are saved into Postgres in a single atomic transaction.Eventual Consistency: A background worker polls the outbox table and pushes pending events to Kafka.🛠️ Core Components1. Kafka Client (infrastructure/kafka.client.js)Manages the lifecycle of the Kafka producer. It includes a Safety Shield to prevent system crashes during the connection handshake.connectKafka(): Initializes the TCP handshake with the broker.getProducer(): A bulletproof getter that returns a "Mock" producer if the real one isn't ready, preventing is not a function errors.2. Event Publisher (infrastructure/messaging/publisher.js)The primary interface for services to broadcast events.JavaScriptimport { publishEvent } from '@yourorg/shared-kernel';

await publishEvent('order-created', { orderId: 12, amount: 150.00 });

🧪 Development & TroubleshootingCommon Error: KAFKA_NOT_INITIALIZEDIf you see this in the logs, it means an order was placed before the Kafka handshake finished.Don't panic: The order is safe in the DB. The Outbox Worker will automatically sync this event once the connection is stable.Deployment (Docker)To apply infrastructure changes and clear the module cache:Bashdocker compose up -d --build node-shared-kernel
